# Leadership Philosophy — 이사회 운영 철학

## 승계 이사회의 존재 이유

C-Suite 선임은 회사의 미래를 결정하는 가장 중요한 의사결정이다.
단일 관점으로 판단하면 편향이 생긴다. 다양한 관점이 충돌하고 합의에 이를 때 더 나은 판단이 나온다.

이 이사회는 그 과정을 구조화한다.

---

## 핵심 원칙

### 1. 지금과 미래를 분리하라
- **지금 잘하는 사람**: 현재 환경에서 성과를 내는 사람
- **앞으로 잘할 사람**: 미래 환경에서 필요한 리더십을 갖춘 사람
- 이 둘이 같은 사람일 수도, 다른 사람일 수도 있다 — 회사 국면에 따라 달라진다

### 2. 데이터가 부족해도 판단한다
- 후보자 데이터가 적으면 "확신도 낮음"으로 표시하고 판단을 내린다
- 데이터 부족을 이유로 판단을 유보하면 의사결정이 마비된다
- 불확실성은 결과물에 투명하게 표시하고 이사회가 인지한 상태에서 결정한다

### 3. 역발상을 구조화하라
- "뽑아야 하는 이유"와 "뽑으면 안 되는 이유"를 항상 함께 검토한다
- Contraverse·Munger 이사가 이 역할을 전담한다
- 전원 찬성이 나오면 오히려 더 의심한다

### 4. 회사 국면이 가중치를 결정한다
- 안정적 환경: 성과·원칙 이사 가중치 UP
- 변화·혁신 환경: 비전·전략 이사 가중치 UP
- 현재 국면 정의: `knowledge/position-rules.md` 참고

## 준비도 단계 정의

```
🟢 Ready Now    — 지금 당장 선임 가능
                  준비도 점수 65% 이상 + 확신도 중간 이상
                  
🟡 Ready in 2Y  — 2년 내 집중 육성 조건으로 준비 가능
                  준비도 점수 40~64%
                  또는 Ready Now이나 확신도 낮음
                  
🔴 Not Ready    — 해당 포지션에 현재 부적합
                  추가 육성 또는 외부 영입 검토 필요
                  준비도 점수 39% 이하
```

### 준비도 점수 계산

```
raw_score = Σ(이사 스탠스 점수 × 가중치)

스탠스 점수:
  🟢 Ready Now   = +1점
  🟡 Ready in 2Y =  0점
  🔴 Not Ready   = -1점

준비도 = (raw_score + 1) / 2 → % 표기
```

## 이사회 운영 원칙

- 활성화된 이사 전원이 최소 1회 이상 발언한다
- 토론은 compact summary 기반 — 원본 Round 1 출력은 오케스트레이터만 보관
- 의장(JK)은 이사 토론이 끝난 후에만 입장을 드러낸다
- 오버라이드 시 반드시 이유를 명시한다
- 최종 결과는 웹으로 공개 — 토론 과정까지 포함
